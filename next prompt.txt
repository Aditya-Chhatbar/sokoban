## Goal

- Keep macro push states for cycle pruning and solver correctness.
- Optimize the solver using actual player walking distance, not push count.
- Treat every single player step as one micro move.
- Make the move counter and hints reflect total micro moves.
- Preserve the existing shorthand interaction model, but count the cost of the full path taken by the player, including long clicks or teleports that traverse multiple cells.

## Approach

- Model the solver as a weighted search over macro Sokoban states.
- Each transition represents a push, and each edge weight is the number of player micro moves required to reach and execute that push.
- Use that weighted cost for search ordering and for the final solution cost.
- Keep macro-state pruning to avoid cycles.
- Use the resulting path to drive hint playback and move counting in the UI, so both reflect micro-move distance rather than push count.
